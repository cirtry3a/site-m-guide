# site-m-guide

## 项目简介

`site-m-guide` 是一个以归档和发布多个独立 HTML 页面为目的的仓库。  
本仓库不针对任何单个域名或具体网站，仅作为一组独立 HTML 文件的汇集与呈现空间。  
通过 GitHub Pages 或其他静态托管方式，可将这些页面公开访问，便于查阅、分享或记录。

## 目录说明

```
site-m-guide/
├── docs/               # 存放所有 HTML 页面及相关资源
│   ├── page-a.html     # 示例页面 A
│   ├── page-b.html     # 示例页面 B
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   ├── script.js
│   └── images/
├── README.md           # 本文件
└── .gitignore          # Git 忽略规则
```

- **docs/**：核心目录，每个 HTML 文件对应一个独立页面。  
- **assets/**：存放页面共享的样式、脚本和媒体资源，便于复用和维护。  
- **README.md**：仓库说明文件。  
- **.gitignore**：配置不需要纳入版本控制的文件或目录。

## 页面归档说明

本仓库中的每个 HTML 文件均为独立、自包含的页面，可单独打开或部署。  
页面内容不限于特定主题，可涵盖笔记、工具、演示、存档等多种用途。  

归档原则：
- 每个页面应有清晰的文件命名，便于识别。
- 如需引用外部资源（如 CDN 库），请在页面内注明来源。
- 鼓励在页面头部添加简要注释，说明页面用途或版本信息。

## 使用方式

1. **克隆仓库**  
   ```bash
   git clone https://github.com/your-username/site-m-guide.git
   ```

2. **添加新页面**  
   在 `docs/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。

3. **本地预览**  
   直接用浏览器打开 `docs/` 下的 HTML 文件即可查看效果。

4. **发布**  
   若启用 GitHub Pages，可将 `docs/` 设置为发布源，即可通过 `https://your-username.github.io/site-m-guide/` 访问各页面。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面会通过 commit 记录。
- 欢迎提交 Issue 或 Pull Request 建议改进，但请注意保持内容中立、不涉及具体推广。
- 对于不再需要的页面，建议先归档至独立分支或删除，并更新目录说明。
